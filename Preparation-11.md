# **Agile & Project Management — Complete Breakdown**

---

# 🟦 **1. Agile (Mindset & Methodology)**  
Agile is a **philosophy** focused on delivering value quickly through iterative development and continuous feedback.

### Core Agile Principles
- Deliver working software frequently  
- Welcome changing requirements  
- Close collaboration between teams  
- Continuous improvement  
- Sustainable pace  

### Why DevOps cares
- CI/CD aligns perfectly with Agile  
- Frequent releases require automation  
- DevOps + Agile = Faster, safer delivery  

---

# 🟧 **2. Scrum (Agile Framework)**  
Scrum is the most widely used Agile framework.

### Scrum Roles
- **Product Owner** → Defines priorities & requirements  
- **Scrum Master** → Facilitates process, removes blockers  
- **Development Team** → Engineers, DevOps, QA  

### Scrum Ceremonies
- **Sprint Planning**  
- **Daily Standup**  
- **Sprint Review**  
- **Sprint Retrospective**  

### Scrum Artifacts
- Product Backlog  
- Sprint Backlog  
- Increment  

---

# 🟩 **3. JIRA (Project & Issue Tracking)**  
JIRA is the most widely used Agile project management tool.

### What JIRA is used for
- User stories  
- Tasks & sub-tasks  
- Bugs  
- Sprint boards (Scrum/Kanban)  
- Release tracking  
- Dashboards & reporting  

### Why DevOps uses JIRA
- Tracks CI/CD work  
- Tracks incidents & changes  
- Integrates with GitHub, GitLab, Jenkins  

---

# 🟪 **4. Confluence (Documentation & Knowledge Base)**  
Confluence is used for **team documentation**.

### What Confluence is used for
- Architecture diagrams  
- Runbooks  
- SOPs  
- RCA documents  
- Project documentation  
- Meeting notes  

### Why DevOps uses it
- Store deployment guides  
- Document Kubernetes clusters  
- Share troubleshooting steps  

---

# 🟥 **5. ServiceNow (ITSM Platform)**  
ServiceNow is the most widely used **IT Service Management** platform.

### What ServiceNow provides
- **Incident Management**  
- **Change Management**  
- **Problem Management**  
- **CMDB**  
- **Service Catalog**  

### Why DevOps uses it
- Auto-create incidents from monitoring alerts  
- Manage production changes  
- Track outages and SLAs  

---

# 🟦 **6. Sprint Planning**  
Sprint Planning defines **what the team will deliver** in the next sprint.

### Inputs
- Product backlog  
- Team capacity  
- Priorities  

### Outputs
- Sprint backlog  
- Sprint goal  
- Acceptance criteria  

### Why DevOps participates
- Plan CI/CD work  
- Plan infrastructure tasks  
- Plan automation improvements  

---

# 🟧 **7. RCA (Root Cause Analysis)**  
RCA identifies the **true underlying cause** of an incident.

### RCA Steps
1. What happened  
2. Why it happened  
3. Root cause  
4. Corrective actions  
5. Preventive actions  

### Tools used
- 5 Whys  
- Fishbone diagram  
- Timeline analysis  

### Why DevOps/SRE uses RCA
- Prevent repeated outages  
- Improve reliability  
- Strengthen CI/CD pipelines  

---

# 🟩 **8. Incident Management**  
Incident Management restores service **as quickly as possible**.

### Incident Lifecycle
1. Detection (monitoring alert)  
2. Logging (ServiceNow ticket)  
3. Triage (severity assignment)  
4. Investigation  
5. Mitigation  
6. Resolution  
7. RCA  
8. Closure  

### Severity Levels
- **Sev1** → Critical outage  
- **Sev2** → Major degradation  
- **Sev3** → Minor issue  

### Why DevOps/SRE cares
- On-call rotations  
- SLO/SLA/SLI reliability metrics  
- Automated remediation  

---

# **How These Fit Together in Real DevOps Workflows**

```
Agile → Scrum → JIRA (stories, tasks)
        ↓
Sprint Planning → DevOps tasks → CI/CD work
        ↓
Incidents → ServiceNow → RCA → Confluence documentation
        ↓
Continuous Improvement → Next Sprint
```

### Example Workflow
1. Product Owner creates stories in JIRA  
2. Team selects stories during Sprint Planning  
3. DevOps builds CI/CD pipelines  
4. Incident occurs → ServiceNow ticket  
5. RCA documented in Confluence  
6. Improvements added to next sprint  

This is exactly how modern engineering teams operate.
