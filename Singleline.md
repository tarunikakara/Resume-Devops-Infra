# ⭐ **1–2 Line Interview Answers for ALL Topics**

---

# **Programming & Scripting**
- **Python** – I use Python for automation, cloud scripting (boto3), and data tasks like ETL or Lambda functions.  
- **Java** – Java is commonly used for backend microservices that we containerize and deploy on Kubernetes.  
- **Bash/Shell** – Bash helps automate Linux tasks like log rotation, deployments, and CI/CD steps.  
- **Groovy** – Groovy is mainly used to write Jenkins pipelines (Jenkinsfile) for CI/CD automation.  
- **PowerShell** – PowerShell automates Windows and Azure tasks like AD user creation or VM management.  
- **YAML** – YAML is the backbone of Kubernetes manifests, GitHub Actions, Helm charts, and IaC configs.

---

# **Cloud Platforms**
### **AWS**
- **EC2** – Virtual servers used for apps, Jenkins, or bastion hosts.  
- **S3** – Object storage for logs, backups, Terraform state, and static websites.  
- **RDS** – Managed SQL databases like PostgreSQL or MySQL.  
- **Lambda** – Serverless compute for event-driven automation.  
- **CloudWatch** – Central monitoring for logs, metrics, and alerts.  
- **EKS** – Managed Kubernetes for microservices.  
- **KMS** – Encryption for S3, EBS, RDS, and secrets.  
- **CloudFormation** – AWS-native IaC for provisioning resources.

### **Azure**
- **AKS** – Azure’s managed Kubernetes service.  
- **ACR** – Private container registry for AKS images.  
- **Azure DevOps** – CI/CD pipelines and repo management.  
- **Functions** – Serverless compute for event-driven tasks.  
- **Monitor** – Metrics, logs, and alerts for Azure workloads.  
- **Azure AD** – Identity and RBAC for Azure resources.... (azure active directory)

### **OpenStack**
- Private cloud platform used to build on‑premise cloud environments similar to AWS.

---

# **IaC & Automation**
- **Terraform** – I use Terraform to provision multi-cloud infrastructure in a repeatable, version-controlled way.  
- **Ansible** – Automates server configuration like installing packages or setting up environments.  
- **CloudFormation / ARM / Bicep** – Cloud-native IaC for AWS and Azure deployments.  
- **Packer** – Builds golden VM or container images with pre-installed software.  
- **Jenkins / GitHub Actions** – Automate build, test, and deployment pipelines.  
- **ArgoCD** – GitOps tool that syncs Kubernetes clusters with Git.

---

# **CI/CD & DevSecOps**
- **Jenkins / CloudBees / GitLab CI / GitHub Actions / Azure DevOps** – Automate build → test → deploy workflows.  
- **AWS CodePipeline** – AWS-native CI/CD for serverless and container apps.  
- **Nexus** – Stores artifacts like Docker images or Maven packages.  
- **SonarQube** – Scans code for bugs, vulnerabilities, and code smells.  
- **Trivy / Snyk / Anchore** – Scan Docker images and dependencies for vulnerabilities.  
- **Vault** – Securely stores and injects secrets into pipelines or Kubernetes.  
- **Checkov / OPA** – Enforce security policies on Terraform and Kubernetes.  
- **AWS SAM CLI** – Build and deploy serverless applications.

---

# **Containerization & Orchestration**
- **Docker / Podman** – Package applications into portable containers.  
- **Docker Buildx** – Build multi-platform images (ARM + x86).  
- **Kubernetes (EKS/AKS/OpenShift)** – Orchestrates containers with scaling, self-healing, and rolling updates.  
- **Helm** – Package manager for Kubernetes apps using charts.  
- **ArgoCD / FluxCD** – GitOps tools that auto-sync Kubernetes with Git.  
- **Docker Swarm** – Simple container orchestrator for small clusters.  
- **Istio** – Service mesh for traffic control, mTLS, and canary deployments.  
- **GitOps** – Git is the source of truth; clusters auto-sync from Git.  
- **Ingress** – Exposes Kubernetes services externally.  
- **HPA** – Auto-scales pods based on CPU, memory, or custom metrics.

---

# **Security & Compliance**
- **Vault** – Centralized secrets management with dynamic credentials.  
- **AWS IAM / Azure AD** – Identity and access control for cloud resources.  
- **AWS Config / Azure Policy** – Enforce compliance rules like encryption or tagging.  
- **FedRAMP / SOC2 / ISO 27001 / PCI-DSS** – Compliance frameworks for security and governance.  
- **Policy-as-Code** – Automate security checks using OPA or Checkov.  
- **Vulnerability Management** – Scan and fix security issues in code, images, and cloud configs.

---

# **Monitoring & Observability**
- **Prometheus** – Collects metrics from Kubernetes and apps.  
- **Grafana** – Visualizes dashboards for metrics and alerts.  
- **ELK** – Centralized logging and search for troubleshooting.  
- **Datadog / Splunk** – Full-stack observability for logs, metrics, and traces.  
- **CloudWatch / Azure Monitor** – Cloud-native monitoring and alerting.  
- **ServiceNow ITSM** – Manages incidents, changes, and problem tickets.

---

# **Databases & Data Platforms**
- **Redshift / Snowflake / Databricks** – Cloud data warehouses for analytics.  
- **DynamoDB / MongoDB** – NoSQL databases for high-scale or flexible schemas.  
- **PostgreSQL / MySQL / Oracle / MS SQL** – Relational databases for transactional workloads.

---

# **Data & MLOps**
- **AWS Glue** – Serverless ETL for data transformation.  
- **EMR** – Big data processing using Spark/Hadoop.  
- **Airflow** – Orchestrates ETL and ML pipelines.  
- **SageMaker** – Train, deploy, and monitor ML models.  
- **Dockerized ML Pipelines** – Package ML models into containers for Kubernetes deployment.

---

# **Deployment Strategies**
- **Blue/Green** – Two environments; switch traffic instantly for zero downtime.  
- **Canary** – Release to a small % of users first, then gradually increase.  
- **Rolling** – Replace pods gradually without downtime.  
- **Immutable** – Deploy new servers/images instead of updating existing ones.

---

- **Linux (RHEL/Ubuntu)** – Primary OS for servers, containers, and Kubernetes nodes.  
- **Windows Server** – Used for AD, .NET, and enterprise workloads.  
- **macOS** – Developer environment for Docker, Git, and Kubernetes.  
- **Git / GitLab / Bitbucket** – Version control and CI/CD integration.

# **SRE Practices**
- **SLI/SLO** – Measure and define service reliability targets.  
- **Error Budgets** – Allowed downtime before slowing releases.  
- **Incident Automation** – Auto-remediation and alerting workflows.  
- **RCA Management** – Prevent repeated failures.  
- **Postmortem Analysis** – Blameless documentation after incidents.
